What is HTML?

HTML is Hyper Text Markup Language

It gives structure and meaning to a website

Elements are represented by tags

HTML (Hyper Text Markup Language) documents all have the same basic structure:

        <!DOCTYPE html>
        <html>
          <head>
          </head>
          <body>
          </body>
        </html>

The head contains metadata, which isn't displayed on the page. This includes the title of the page (defined within title tags), which is displayed on the page's tab.

The body defines the contents of the HTML document, including hyperlinks, headings, paragraphs, lists, and images.

HTML is comprised of sets of descriptive tags which define the content on the page.

An HTML element is a pair of tags contains an opening tag, whose element name is between angle brackets, and a closing tag, which is the same name between angle brackets with a slash after the opening bracket, with content between the opening and closing tags.

Most elements contain opening and closing tags, but some contain only one tag.

HTML Tags

HTML tags help to define elements.

A set of HTML tags looks like this:

        <p></p>

An HTML element looks like this:

        <p>This is a paragraph.</p>

Heading Tags

Heading tags define section headings.

h1 is the highest, and largest, section level, and should only be used once on a page.

h6 is the lowest, and smallest, section level. Headings require an opening and closing tag.

A heading element would be defined like this in an html document:


            <h1>This is my first heading</h1>

And would render like this:


            
This is my first heading

Heading elements have predefined whitespace, font style and size, and paragraph breaks.

Paragraph Tags

Paragraph tags define blocks of text and have space before and after them.

A paragraph element would be defined like this in an html document:


            <p>This is my first paragraph.</p>

And would render like this:


            
This is my first paragraph.

Div Tags

Div tags define a section of an HTML document, and are often used to add JavaScript or as containers for CSS styling.

A div element would be defined like this in an html document:


            <div>This is a div</div>

And would render like this:


            This is a div

Lists

An ordered, or numbered, list would be defined in an html document like this:

The HTML tags we've already seen (h1-h6, p, div, ol,ul, and li) all have opening and closing tags. Some HTML elements, however, have a less straightforward syntax.

Anchor Elements

The anchor tag allows you to create links on your page. In this element, you'll include an attribute telling the browser where to link to. The attribute is included in the opening tag and includes a name and a value.

In an anchor element, the href attribute is used to define the destination of the link. The name of the attribute is href, and the value is the link. Clickable text will be contained between the opening and closing tags of the element.

An anchor element is defined like this in an html document:


            <a href="http://www.google.com">Google</a>

And would render like this:


            Google

Image Elements

Image tags, which allow you to insert images into your html document, are not containers for content, and image elements have only one tag.

Image elements contain several attributes, which allow for configuration. The src attribute tells the browser where to find the image that should be displayed. The alt attribute specifies alternate text that describes the image.

An image element is defined like this in an html document:


          <img src="http://placecorgi.com/180" alt="a corgi" />

What does CSS stand for?

Cascading Style Sheets

CSS is a style sheet language that lets you style the elements on your page.

CSS works with HTML, but is not HTML itself.

body {
    background-color: #4a69bd;
    color: #ffffff;
    font-family: 'Barlow', sans-serif;
    margin: 0;
    text-align: center;
}

h1 {
    text-transform: uppercase;
    font-size: 60px;
}

h2 {
    font-size: 40px;
}

A block of CSS code is a rule. The rule starts with a selector.
It has sets of properties and values.

CSS Selectors

So far we have been selecting HTML elements, but what if we want to be more specific?

Example: What if I want to make one of my headers purple but the rest should stay blue?

Remember HTML classes?

<h1 class="main-header">My Favorite Things</h1>

We can use these to apply CSS styles

When using a class selector,
make sure you put a period in front of the class name

.main-header {
    color: blue;
    font-size: 40px;
}

This style will only apply to elements with the class name of "main-header"

What will happen if two elements have the same class name?

Lets practice using class names to apply styles!

We can also use HTML ids for CSS

<h1 id="fav-things">My Favorite Things</h1>

When using an ID selector,
make sure you put a hash in front of the ID name

#fav-things {
    color: blue;
    font-size: 40px;
}

This style will only apply to the single element with the id name of "fav-things"

Remember, an ID can only be one element

Lets practice using ids to apply styles!

The CSS Cascade

What do you think will happen here? Will the H1 elements be blue or green?

h1 {
    color: blue;
}

h1 {
    color: green;
}

ANSWER: green

Since CSS is "cascading", it starts at the top of the stylesheet and moves down.

So, the last declaration will (almost) always win

!important

We can use the !important tag to make sure a style gets applied.

h1 {
    color: blue!important;
}

h1 {
    color: green;
}

In this situation, the cascade would be ignored, and the H1 would be blue

Its not best practice to use important tags; they can make our code hard to understand.
